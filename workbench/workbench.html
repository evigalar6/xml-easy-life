XML Easy Life Help

Use this guide as a fast checklist for the tabbed workflow: Inspector, XPath, XSD, and XSLT.

1. Quick Start

  1. Upload XML or load XML from current tab.
  2. Open Inspector to format, validate, and review summary.
  3. Open XPath to generate and run expressions.
  4. Open XSD to run basic schema checks.
  5. Open XSLT to transform and inspect output.

2. Inspector Tab

  • Format XML toggles between formatted and original source.
  • Validate checks XML well-formedness.
  • Re-scan XML refreshes structure summary (root, element count, namespaces, size).
  • Error navigation appears only when validation finds parser issues.

3. XPath Tab

  • Manage namespace mappings as prefix=uri.
  • Use Detect Namespaces From XML to prefill prefixes.
  • Generate XPath suggestions from current cursor location.
  • Run XPath and copy full results from the results panel.

5. Generation Rules

  • Needs well-formed XML.
  • Uses current cursor position.
  • Infers context from open tag stack before cursor.

6. Ambiguity Cases

  • Cursor on whitespace between sibling elements.
  • Cursor before root element or after document end.
  • Invalid XML.

In ambiguous cases, generator may fall back to a nearby or root element.

7. XSD Tab

  • Upload XSD and run lightweight structural validation.
  • Checks include root compatibility and required direct child elements.

8. XSLT Tab

  • Upload/paste stylesheet and run transform.
  • The extension uses a JS-based XSLT-lite engine for supported templates.
  • Transform output has its own copy button.

9. Input Area Notes

  • Editor is internally scrollable with synced line numbers.
  • Recent XML history is available from the dropdown.
  • Tabs are task-focused and keep state while switching.
