XML Easy Life Help

Use this guide as a fast checklist while testing XML, XPath, XSD, and XSLT workflows.

1. Quick Start

  1. Upload XML or load XML from current tab.
  2. Use Validate to check well-formedness.
  3. Run XPath queries in the tester panel.
  4. Use Generate XPath From Cursor for suggestions.
  5. Use XSLT Preview to transform and inspect output.

2. XPath Suggestions

  • Absolute â€¢ Fragile: exact indexed path. Precise but breaks if structure changes.
  • By tag â€¢ Broad: matches all nodes with tag name.
  • By attribute â€¢ Stable: best when attribute is unique.
  • By text â€¢ Medium: useful but text changes can break it.

3. How Generation Works

  • Needs well-formed XML.
  • Uses current cursor position.
  • Infers context from open tag stack before cursor.

4. Namespaces

  • Add mappings as prefix=uri, one per line.
  • Use Detect Namespaces From XML to prefill prefixes.
  • If XML has a default namespace, use prefixed names in XPath.

5. Ambiguity Cases

  • Cursor on whitespace between sibling elements.
  • Cursor before root element or after document end.
  • Invalid XML.

In ambiguous cases, generator may fall back to a nearby or root element.

6. Validation and Transform Notes

  • Editor is internally scrollable with line numbers.
  • Format button toggles between formatted and original XML.
  • Error controls appear only after validation errors are found.
  • Recent XML history is available from the dropdown.
  • XSD validation uses lightweight structural checks.
  • XSLT output has its own copy button for quick sharing.
